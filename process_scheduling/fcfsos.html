About FCFS

Algorithm overview, characteristics and trade-offs

First-Come, First-Served (FCFS) is the simplest CPU scheduling algorithm where processes are executed in the order they arrive in the ready queue. It is non-preemptive — once a process starts executing, it runs to completion.

Key Characteristics

  • Non-preemptive: once a process gets the CPU it runs to completion
  • Processes execute strictly in order of arrival time
  • Simple to understand and implement
  • Backed by a FIFO queue

Advantages

  • Simplicity: easy to understand and implement
  • Fair in the sense that processes run in arrival order
  • No starvation: every process eventually gets CPU time
  • Zero overhead for process selection logic

Disadvantages

  • Prone to the "convoy effect" — short processes wait behind long ones
  • Can result in poor average waiting time
  • Not suitable for interactive or real-time systems
  • No priority consideration for time-sensitive tasks
