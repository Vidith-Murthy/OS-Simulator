Memory Management

Page Replacement
Algorithms

Page replacement algorithms are techniques used in operating systems to manage memory efficiently when physical memory is full. When a new page needs to be loaded and no free frame is available, these algorithms determine which existing page to replace.

// SELECT ALGORITHM

Demand Paging Concept

Pages are loaded into physical memory only when requested by the running program — the foundation of all page replacement strategies.

FIFO Basic

First-In, First-Out replaces the oldest page in memory. Simple to implement but susceptible to Bélády's Anomaly.

Simulate

Optimal Theoretical

Replaces the page not used for the longest time in the future. Produces minimum page faults — the benchmark algorithm.

Simulate

LRU Practical

Least Recently Used evicts the page unused for the longest time. Based on temporal locality — not susceptible to Bélády's Anomaly.

Simulate

MRU Variant

Most Recently Used replaces the page accessed most recently — effective when recently used pages are unlikely to be reused soon.

Simulate

LFU Frequency

Least Frequently Used replaces the page with the fewest accesses — assumes rarely accessed pages are least likely to be needed.

Simulate

Random Simple

Selects a page at random for replacement with no consideration of access history — extremely simple, unpredictable performance.

Simulate

Second-Chance Clock

An enhanced FIFO that gives recently-used pages a "second chance" via a reference bit — also known as the Clock algorithm.

Simulate

MFU Frequency

Most Frequently Used replaces the most-accessed page — assumes highly accessed pages have completed their usage cycle.
