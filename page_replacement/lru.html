LRU Page Replacement

Evicts the page that has not been accessed for the longest period — the most widely used practical page replacement algorithm.

How It Works

LRU keeps track of when each page was last accessed. When a page fault occurs and all frames are full, the algorithm selects the page that hasn't been used for the longest time and replaces it with the new page. This approach uses past usage patterns as a predictor of future accesses, exploiting the principle of temporal locality.

Advantages

  • Not susceptible to Bélády's Anomaly
  • Effectively identifies pages with minimal recent usage
  • Often outperforms FIFO in real-world workloads
  • Well-studied with provable performance guarantees
  • Widely implemented in modern operating systems

Disadvantages

  • Requires sophisticated data structures to track page usage history
  • Demands hardware assistance for efficient implementation
  • Higher operational overhead due to timestamp tracking
  • More complex to implement than FIFO
  • Full LRU is expensive — approximations are typically used
