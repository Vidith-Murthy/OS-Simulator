Linked File Allocation

Linked File Allocation stores file blocks anywhere on disk. Each block contains a pointer to the next block in the file.

The directory entry stores the first block, and the system follows the pointer chain to read the whole file.

This removes external fragmentation, but random access is slow because the system may need to traverse many pointers.

Advantages

  • No external fragmentation.
  • Files can grow easily.
  • Free blocks can be used from anywhere on disk.
  • No compaction is required.

Disadvantages

  • Random access is inefficient.
  • Pointers consume extra space.
  • If a pointer is corrupted, file data can be lost.
  • Sequential access may be slower than contiguous allocation.
