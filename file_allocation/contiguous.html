Contiguous File Allocation

Contiguous File Allocation stores each file in a set of consecutive disk blocks. The directory only needs the starting block and length of the file.

Because all blocks are placed together, sequential access is very fast and direct access is also simple: block i can be found using start block + i.

The main problem is external fragmentation. After many files are created and deleted, enough free blocks may exist but not in one continuous run.

Advantages

  • Simple and easy to implement.
  • Fast sequential access and direct access.
  • Very little metadata is needed.
  • Good disk I/O because nearby blocks reduce seek time.

Disadvantages

  • External fragmentation can waste disk space.
  • File growth is difficult if next blocks are occupied.
  • File size is better known at creation time.
  • Compaction may be required.
