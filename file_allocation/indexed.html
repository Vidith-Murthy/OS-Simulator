Indexed File Allocation

Indexed File Allocation gives each file an index block. The index block stores addresses of all data blocks of that file.

To access a file block, the operating system reads the index block and jumps directly to the required data block.

This supports direct access and file growth, but it requires extra space for index blocks.

Advantages

  • No external fragmentation.
  • Supports direct access to any block.
  • Files can grow dynamically.
  • Better for frequently modified files.

Disadvantages

  • Extra index block overhead.
  • Small files may waste index space.
  • Very large files may need multilevel indexing.
  • Index block access can add extra I/O.
